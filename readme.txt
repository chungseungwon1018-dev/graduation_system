프로젝트 개요: 졸업학점 관리 시스템
목적:Python + MySQL 기반의 웹 시스템으로, 학생은 자신의 학점 이수 현황을 업로드된 .xlsx 파일을 통해 자동 분석받고, 관리자는 졸업 요건 기준에 따른 학생들의 이수 상태를 효율적으로 관리할 수 있는 시스템을 개발합니다.

개발환경
백엔드: Python (Flask 또는 FastAPI)
프론트엔드: HTML/CSS + JS (추후 React로 이식 가능)
DBMS: MySQL
파일처리: openpyxl 또는 pandas로 .xlsx 파싱

배포환경: 로컬 테스트 → 클라우드 서비스 연동 가능 (예: Render, AWS)

기능 요구사항
1. 학생 사용자 기능
1.1 회원 정보 관리 (파일 업로드 기반)
사용자 .xlsx 파일에서 다음 필드를 자동 추출 후 DB에 저장:

개인정보 필드: 대학, 학과, 전공, 부전공, 다전공, 과정, 입학일자, 학번, 성명, 교과 적용년도, 이수학기, 생년월일, 학년, 평생사제상담 건수
이수학점 필드: 구분, 영역, 세부영역, 년도, 학기, 교과목번호, 교과목명, 학점, 이수구분, 성적
수정 가능하며, 실시간 DB 반영 필요

1.2 졸업요건 자동 분석
업로드된 학점 데이터를 기준으로, 사전에 등록된 졸업 요건과 비교하여 충족 여부 분석
분석 로직은 모듈화 필요 (입학년도, 학과, 전공 기준으로 요건 매핑)

1.3 대시보드
분석 결과 시각화: 항목별 충족률 표 + 차트 제공
부족 요건 강조 색상 적용 (예: 빨간색 + 투명도 감소)
예측 수강 추천 로직 기본 탑재 (예: 부족 영역에 대한 교과목 추천)

1.4 알림 확인
관리자가 발송한 메시지를 로그인 시 확인 가능
읽음/안읽음 상태 기록

2. 관리자 기능
2.1 졸업요건 등록/수정/삭제
연도 및 학과별 졸업 기준 CRUD 기능
영역 구분, 이수 학점, 필수 과목 등 상세 규칙 저장 가능

2.2 학생 이수현황 조회
조건 필터링 (학과/학년/입학연도 등) 기반 이수 데이터 검색
학생 상세 페이지에서 개인 분석 결과 확인

2.3 알림 전송
개별 또는 그룹(필터 조건 적용) 대상 맞춤형 메시지 발송
알림 히스토리 로그 저장

2.4 통계 대시보드
집단별 충족률 통계, 미충족 항목 비율 시각화
학과/연도별 경향 분석용 차트 제공

비기능 요구사항
보안 : 개인정보 암호화(예:AES), 역할기반 접근제어(RBAC:학생/관리자)
신뢰성 : 분석 알고리즘 검증, 예외 처리 및 자동 롤백
성능 : 100명 이상 동시 사용자 대응
사용성 : 접근성 준수한 UI, 직관적 대시보드 UX
유지보수 : 졸업 요건 동적 관리 가능, 변경사항 자동 반영 구조
추적 기능 : 사용자 로그 기록 : 로그인, 수정, 알림 열람 등
권한 제어 : 관리자 기능 접근 제한, 학생은 본인 데이터만 열람 가능
모듈화 : 다음과 같은 모듈로 분리
xlsx_parser_module.py: 파일 파싱 및 필드 추출
graduation_requirements_checker.py: 요건 매칭 분석
notification_system.py: 알림 발송 및 열람
dashboard_renderer.py: 시각화 출력

페이지 구조
1. 로그인 페이지
ID/PW 기반 로그인
계정 생성: 학번 기반 ID, 이름, 소속 입력
관리자 여부 체크박스
실패 시 오류 메시지

2. 마이페이지
개인정보 수정 폼
내 이수 데이터 및 상태 조회

3. 졸업학점 분석 페이지
요건별 충족률 차트 제공
충족/미달 항목 별도 시각 강조
캡스톤/논문/상담 요건 상태도 포함

4. 관리자 페이지
졸업요건 관리 (추가/수정/삭제)
학생 목록 및 상세 조회
알림 작성 및 발송 UI
통계 대시보드 (필터 선택 포함)
